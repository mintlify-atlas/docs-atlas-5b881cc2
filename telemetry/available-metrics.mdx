# Available Metrics Catalog

This page documents all telemetry metrics available from Nokia SROS and SR Linux devices in the BNG lab.

## Metric Overview

Metrics are collected via gNMI streaming telemetry at 5-second intervals and exported to Prometheus.

**Metric Types**:
- **Counter**: Cumulative value that only increases (resets on restart)
- **Gauge**: Value that can increase or decrease
- **String**: Converted to labels in Prometheus

**Naming Convention**:
- SROS: `/state/` prefix removed by gNMIc processor
- SR Linux: YANG path structure preserved
- Underscores separate path elements

## SROS Metrics (BNG1, BNG2)

Metrics from Nokia Service Router Operating System devices.

### Port Statistics

Physical interface metrics from `/state/port/statistics/` and `/state/port/ethernet/statistics/`

<Tabs>
  <Tab title="Traffic Counters">
| Metric | Type | Description |
|--------|------|-------------|
| `port_statistics_in_packets` | Counter | Total ingress packets |
| `port_statistics_out_packets` | Counter | Total egress packets |
| `port_statistics_in_octets` | Counter | Total ingress bytes |
| `port_statistics_out_octets` | Counter | Total egress bytes |

**Labels**: `device`, `port_id`

**Example Query**:
```promql

rate(port_statistics_in_octets{device="bng1"}[5m]) * 8

rate(port_statistics_in_packets{device="bng1",port_id="1/1/c1/1"}[5m])
```

### System Metrics

Device-level resource usage from `/state/system/`

<Accordion title="CPU Utilization">
| Metric | Type | Description |
|--------|------|-------------|
| `system_cpu` | Gauge | CPU utilization percentage (0-100) |

**Labels**: `device`, `sample_period`

**Example Query**:
```promql
# Current CPU usage
system_cpu{device="bng1"}

# Average CPU across all BNGs
avg(system_cpu{device=~"bng.*"})

# Max CPU in last hour
max_over_time(system_cpu{device="bng1"}[1h])
```
</Accordion>

## Common Query Patterns

### Traffic Analysis

<CodeGroup>
```promql Bits per Second
# SROS port traffic
rate(port_statistics_in_octets[5m]) * 8

# SR Linux interface traffic
rate(interface_statistics_in_octets[5m]) * 8
```

```promql Packets per Second
# SROS
rate(port_statistics_in_packets[5m])

# SR Linux
rate(interface_statistics_in_packets[5m])
```

```promql Utilization Percentage
# Assuming 10Gbps interface
(rate(port_statistics_in_octets[5m]) * 8 / 10000000000) * 100
```

```promql Top N Interfaces
# Top 5 by traffic
topk(5, rate(port_statistics_in_octets[5m]))

# Bottom 5
bottomk(5, rate(port_statistics_in_octets[5m]))
```
</CodeGroup>

### Error Monitoring

<CodeGroup>
```promql Error Rate
# Errors per second
rate(port_statistics_in_errors[5m])

# Error percentage
(rate(port_statistics_in_errors[5m]) / 
 rate(port_statistics_in_packets[5m])) * 100
```

```promql Identify Problems
# Any interface with errors
port_statistics_in_errors > 0

# Interfaces with high error rate
rate(port_statistics_in_errors[5m]) > 10
```
</CodeGroup>

### Resource Monitoring

<CodeGroup>
```promql CPU Analysis
# Average CPU
avg(system_cpu)

# Max CPU per device
max by (device) (system_cpu)

# CPU above threshold
system_cpu > 80
```

```promql Memory Analysis
# Memory percentage
(system_memory_pools_in_use / system_memory_pools_available) * 100

# Free memory in GB
system_memory_pools_free / 1073741824
```
</CodeGroup>

### Session Monitoring

<CodeGroup>
```promql Active Sessions
# Total active sessions
sum(subscriber_mgmt_local_user_db_ipoe_session_stats_current)

# Sessions per BNG
sum by (device) (subscriber_mgmt_local_user_db_ipoe_session_stats_current)
```

```promql Session Dynamics
# Setup rate
rate(subscriber_mgmt_local_user_db_ipoe_session_stats_setup[5m])

# Failed sessions
rate(subscriber_mgmt_local_user_db_ipoe_session_stats_setup_failed[5m])

# Success rate
(rate(subscriber_mgmt_local_user_db_ipoe_session_stats_setup[5m]) / 
 (rate(subscriber_mgmt_local_user_db_ipoe_session_stats_setup[5m]) + 
  rate(subscriber_mgmt_local_user_db_ipoe_session_stats_setup_failed[5m]))) * 100
```
</CodeGroup>

## Discovering New Metrics

### List All Available Metrics

```bash
# From Prometheus
curl http://localhost:9090/api/v1/label/__name__/values | jq

# From gNMIc exporter
curl http://localhost:9273/metrics | grep "^[a-z]" | cut -d' ' -f1 | sort -u
```

### Explore Metric Labels

```promql
# All labels for a metric
group by (__name__, device, port_id) (port_statistics_in_packets)

# Unique devices
count by (device) ({__name__=~".+"})

# Unique ports on a device
count by (port_id) (port_statistics_in_packets{device="bng1"})
```

### Find Metrics by Pattern

```promql
# All CPU metrics
{__name__=~".*cpu.*"}

# All BGP metrics
{__name__=~".*bgp.*"}

# All counters (assuming naming convention)
{__name__=~".*_total$"}
```
